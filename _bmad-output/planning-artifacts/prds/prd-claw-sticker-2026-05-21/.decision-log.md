# Decision Log — Claw Sticker Plugin PRD

> Canonical memory and audit trail for the PRD run.

---

## D-001: 发布形态
- **决策**：作为社区插件发布（非内部工具）
- **影响**：需要完整的 manifest、config schema、文档、clawhub 发布流程

## D-002: V1 表情包范围
- **决策**：使用插件内置 7 个预设表情包（happy, love, cool, confused, nervous, sigh, awkward）
- **后续**：V2 考虑用户自定义表情包

## D-003: V1 Channel 范围
- **决策**：仅适配企业微信（WeCom），`MEDIA:` 指令格式
- **后续**：其他 channel 使用不同机制（如 Telegram/Discord 用 `filePath`）

## D-004: 随机性策略
- **决策**：特定语境触发 + 概率控制，追求"像人"的自然感觉。即使场景匹配也不一定每次都发
- **用户原话**："比如如果任务完成，每次都发一个表情，感觉很怪，非常机械"

## D-005: 已有技术方案输入
- sticker-format-guard-plugin-prompt.md — 格式修正的完整规则和代码参考
- sticker-comprehensive-guide.md — 表情包完整指南（场景映射、使用规则、铁律、已知问题）
- OpenClaw Plugin SDK docs — hooks、entry points、packaging 全套

## D-006: 之前插件尝试的问题
- 之前尝试过开发但卡在插件扫描路径（只扫描 `dist/extensions/`）和预编译问题
- 本次需要明确解决打包和安装路径问题

## D-007: 插件命名
- **决策**：插件 ID 为 `claw-sticker`（涵盖表情引导 + 格式修正双能力）

## D-008: V1 配置策略
- **决策**：V1 全部 hardcode，不暴露可配置参数
- **后续**：V2 考虑开放概率、冷却窗口等参数给 operator

## D-009: 冷却机制
- **决策**：纳入 V1 设计，防止连续消息都带表情，增强自然感

## D-010: Prompt 引导注入时机
- **决策**：静态表情指南（列表、映射、格式规则）在会话新建时注入一次，不在每个 turn 重复注入
- **原因**：静态内容每 turn 注入浪费 context window，无意义
- **影响**：只有动态冷却状态才需要 per-turn 级别的注入
